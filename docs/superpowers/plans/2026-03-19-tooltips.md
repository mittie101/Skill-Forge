# Tooltips Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add hover-triggered floating-card tooltips (ⓘ icon) to form fields and action buttons across Generator, Builder, and Install views.

**Architecture:** Pure CSS hover — `.tooltip-wrap:hover .tooltip-card { opacity: 1 }`. No JS, no new files. The ⓘ icon is an `<i class="info-icon">` inside a `<span class="tooltip-wrap">` that also contains a `<div class="tooltip-card">`. Cards open upward by default; the `.below` modifier flips them downward for fields at the top of a panel and all buttons (avoids overflow clipping in scrollable panels).

**Tech Stack:** Vanilla HTML/CSS, Electron 35 renderer (contextIsolation: true, no JS required)

## Reusable snippet reference

Use this pattern everywhere. Only `below`, the `id`, `tooltip-title` text, and body text change per instance:

```html
<!-- Inside .field-header, after the <label>: -->
<span class="tooltip-wrap">
  <i class="info-icon" aria-label="More information">i</i>
  <div class="tooltip-card">
    <div class="tooltip-title">TITLE HERE</div>
    Body text here.
  </div>
</span>

<!-- Add class="below" to .tooltip-card for top-of-panel fields and buttons: -->
<div class="tooltip-card below">…</div>

<!-- For buttons, wrap the entire <button> in .tooltip-wrap: -->
<span class="tooltip-wrap">
  <button …>Label</button>
  <div class="tooltip-card below">…</div>
</span>
```
